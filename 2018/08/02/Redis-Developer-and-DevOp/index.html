第一章 初识Redis

盛赞Redis

Redis(REmote DIctionary Server) 是一种基于键值对（key-value）的NoSQL数据库。
value类型可以是 String, Hash, list, set, zset, Bigmaps,HyperLogLog, GEO.

  1. http://redis.io
  2. http://antirez.com
  3. http://github.com/antirez/redis

Redis特性

速度快，读写速度可以达到10万/秒

原因：

  1. 所有数据存在内存中。
  2. C语言实现。
  3. 单线程架构，预防了多线程竞争。
  4. 代码精打细磨

基于键值对的数据结构服务器

Java 的map，Python 的dict等

丰富的功能

  1. 支持键过期，可以实现缓存。
  2. 提供发布订阅功能，可以实现消息队列。
  3. 支持Lua脚本，可以利用Lua创造出新的命令。
  4. 提供简单的事务支持。
  5. 提供流水线（pipeline）功能，减少网络开销。

简单稳定

早起版本只有2万行代码

客户端语言多

Java, Python, PHP, C, C++, Nodejs

持久化

  1. RDB
  2. AOF

主从复制

支持主从复制，这是分布式Redis的基础。

高可用和分布式

2.8版本提供了高可用实现Redis Sentinel，保证Redis节点的故障发现和故障自动转移。

使用场景

可以做什么

  1. 缓存
  2. 排行榜系统
  3. 计数器应用
  4. 社交网络
  5. 消息队列系统

不可以做什么

  1. 数据规模太大
  2. 冷数据

预备

全局命令

keys *  如果不存在返回 (empty list or set) 如果存在，列出所有的key
dbsize  (integer) 大小
exists [key] 如果存在返回 (integer) 1， 如果不存在返回 (integer) 0
del [key] 如果存在删除key并返回 (integer) 1， 如果不存在返回 (integer) 0
del key1 [key2 ...keyk ] 如果存在删除key并返回的个数 (integer) k， 如果不存在返回 (integer) 0
expire [key] seconds 设置多久之后过期
ttl [key] 返回过期剩余时间。 如果存在返回时间，如果不存在返回-2，如果没有设置过期时间，返回-1
type [key] key的数据结构类型，如果不存在返回nono

数据结构和内部编码

object encoding [key]


单线程架构

字符串

字符串类型的值可以是字符串（简单的字符串，复杂的字符串（如JSON，XML），数字（整数，浮点数），设置是二进制（图片，音频，视频））但是值得最大值不能超过512MB。

命令

时间复杂度

命令 时间复杂度
set key value $O(1)$
get key $O(1)$
del key [key …] $O(k)$ $k$ 是键的个数
mset key value [key value …] $O(k)$ $k$ 是键的个数
mget key [key …] $O(k)$ $k$ 是键的个数
incr key $O(1)$
drcr key $O(1)$
incrby key increment $O(1)$
decrby key decrement $O(1)$
increbyfloat key increment $O(1)$
append key value $O(1)$
strlen key $O(1)$
set range key offset value $O(1)$
getrange key start end $O(n)$ $n$ 是字符串的长度

内部编码

  • int 8个字节的长整型
  • embstr 小于等于39个字节的字符串
  • raw 大于39个字节的字符串

典型的应用场景

  1. 缓存功能 先请求redis，如果有直接返回，如果没有，查找数据库，存到redis中并返回
  2. 计数 自增
  3. 共享session 可以在多个服务器架构的情况下避免用户刷新之后session信息小时，需要重新登录
  4. 限速 通过过期时间设置用户每分钟或者每秒可以请求的次数
