Git-Learn-2

Posted on 2018-07-22

为了不给GitHub带来困扰，这货怎么又新建了一个Test Repository
我决定自己改一个GitLab玩

  1. docker 创建GitLab环境，省略

  2. 在GitLab上创建一个新的Reporsitory， 项目名称为git-learn-test

  3. clone到本地

目前的状态

在本地创建了一个新的文件test_file.txt

民工二连

目前的状态

现在本地比远程仓库多了一个commit，那现在就可以提交给远程了。

此时再打开GitLab网站看，就发现已经更新了，所以现在的状态应该是

现在假设另一个人小B也在维护这个系统。（为了方便，不再下载一遍了，直接在gitlab上改）

修改 test_file.txt文件为

commit message: 2. Added by xiao-A via gitLab.
commit id: 6ab5a9b8

现在的状态

那本地现在怎么办呢？

一种一种情况测试一下

  1. 本地什么都没动, 直接git pull
  1. 本地修改了，但是互相有影响，需要解决冲突

把文件修改为

  1. 本地修改了，但是互相之间没有影响

继续修改gitlab

本地创建新文件 test_file2.txt
Added by local, new file, filename: test_file2.txt

民工二连

这个时候pull会怎么样呢？

经过测试，如果本地什么都没有动，直接pull，没问题
如果本地修改过了，需要先commit
如果commit之后pull有冲突，需要先fix conflict再commit
如果commit之后pull没有冲突，需要为这次merge创建一个新的commit
